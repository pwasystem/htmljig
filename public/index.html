Get Started with HTML Jig

What do you need to know

This project was made to work with Google Firebase.

You need to have Node JS and Firebase Tools installed on your local hardware. This can be a computer, tablet or smartphone.

Remember to create a project with a web application and log in to Firebase Tools before performing the procedures below.

If you're new to Google Firebase, start here:

  1. Install NodeJS from 
     https://nodejs.org/en/download/
  2. Install Google Firebase Tools 
    npm install -g firebase-tools
  3. Log in 
    firebase login
  4. Create a Firebase project 
    firebase init

Then, follow these simple steps to create your website with HTML Jig:

  1. Clone the repository: 
    git clone https://github.com/pwasystem/htmljig.git
  2. Rename the HTMLjig folder: Change the folder name to your Firebase project name.
  3. Modify .firebaserc: Update the project name in the .firebaserc file: 
    {
  "projects": {
    "default": "your_project_name"
  }
}
  4. Configure .env: Replace (your_gemini_api_key) with your actual Gemini API key in the functions/.env file: 
    GEMINIKEY=(your_gemini_api_key)
  5. Update action attribute: Modify the action value in the form of public/jig.html to: 
    action='http://127.0.0.1:5001/(your_project_name)/us-central1/jig'
  6. Install dependencies: Run npm install in the functions directory.
  7. Start Firebase emulator: Run firebase emulators:start.
  8. Access your website: Open http://127.0.0.1:5000/jig.html in your browser.
  9. Fill in the form:
    • File name: Name for your HTML file.
    • Title: Title of your website or company.
    • Description: Information about your website, preferred colors, and desired content.
    • Template: Select a model to use as a template.
    • Images: Provide links to your images.
  10. Click "Create": Wait for the AI to build your website.
  11. View your site: Click on the link to see your creation.
  12. Suggest changes: Use the description field to suggest improvements or publish your website with firebase deploy --only hosting.
